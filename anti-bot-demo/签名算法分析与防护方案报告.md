## 1. 系统概述

### 1.1 防御体系架构

API 签名系统采用 **四层防御体系**：

```
┌─────────────────────────────────────────┐
│           L1: HTTP 请求头指纹            │
│  检测非浏览器客户端、伪造 UA、缺失特征头   │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│           L2: 请求签名验证               │
│  HMAC 绑定路径+请求体+时间戳+nonce        │
│  密钥: VM字节码 + 动态挑战盐              │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│           L3: 设备指纹追踪               │
│  设备 ID 一致性校验、设备劫持检测          │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│           L4: 行为分析引擎               │
│  频率限制 + 间隔规律性 + sigCount 模式    │
└─────────────────────────────────────────┘
```

## 2. VM 字节码架构分析

### 2.1 双层字节码结构

签名引擎采用 **双层字节码虚拟机**：

```
外层字节码 (4字节) → DECRYPT_INNER → EXEC_INNER → RETURN
        ↓                    ↓
    简单初始化        解密字符串池中的加密字节码
                            ↓
                    真正的字节码 (2327字节)
                            ↓
                    执行签名算法逻辑
```

### 2.2 VM 类结构

```javascript
class VM {
  constructor(sboxHex, b64Alphabet, bytecodeHex, strPool, isBrowser) {
    this.sbox = hexToBytes(sboxHex);          // 256字节 S-Box
    this.b64Alphabet = b64Alphabet;           // 64字符自定义 Base64
    this.bytecode = hexToBytes(bytecodeHex);  // 外层字节码 (4字节)
    this.strPool = strPool;                   // 字符串池 (7个元素)
    this.isBrowser = isBrowser;               // 环境标识
    
    // 初始化状态
    this.pc = 0;                              // 程序计数器
    this.stack = [];                          // 执行栈
    this.hashState = [...];                   // 32字节哈希状态
    this.dbgTriggered = false;                // 调试触发标志
  }
  
  run(path, body, ts, nonce, salt) {
    // 签名生成入口
    this.execute(this.bytecode, 0);
    return this.result;
  }
  
  execute(code, startPc) {
    // 字节码执行器
    while (this.pc < code.length) {
      const opcode = code[this.pc];
      this.dispatchOp(opcode);
    }
  }
}
```

### 2.3 操作码定义

VM 支持 **36种操作码**：

| 操作码 | 名称 | 功能 | 示例用途 |
|--------|------|------|---------|
| 0x00 | NOP | 无操作 | - |
| 0x01 | PUSH_CONST | 推送常量 | `stack.push(CONSTANTS[op[1]])` |
| 0x02 | PUSH_STR | 推送字符串池元素 | `stack.push(strPool[op[1]])` |
| 0x03 | POP | 弹出栈顶 | `stack.pop()` |
| 0x04 | DUP | 复制栈顶 | `stack.push(stack[stack.length-1])` |
| 0x05 | SWAP | 交换栈顶两个元素 | `[stack[-1], stack[-2]] = [stack[-2], stack[-1]]` |
| 0x06 | ROT3 | 栈顶三元素轮转 | 旋转操作 |
| 0x07 | ADD | 加法 | `stack.push(a + b)` |
| 0x08 | SUB | 减法 | `stack.push(a - b)` |
| 0x09 | XOR | 异或 | `stack.push(a ^ b)` |
| 0x0A | AND | 与运算 | `stack.push(a & b)` |
| 0x0B | OR | 或运算 | `stack.push(a \| b)` |
| 0x0C | NOT | 非运算 | `stack.push(~a)` |
| 0x0D | SHL | 左移 | `stack.push(a << b)` |
| 0x0E | SHR | 右移 | `stack.push(a >> b)` |
| 0x0F | JMP | 跳转 | `pc = op[1]` |
| 0x10 | JMPZ | 条件跳转（零） | `if (stack.pop() === 0) pc = op[1]` |
| 0x11 | JMPNZ | 条件跳转（非零） | `if (stack.pop() !== 0) pc = op[1]` |
| 0x12 | CALL | 调用子程序 | `execute(innerBytecode, op[1])` |
| 0x13 | RET | 返回 | `return` |
| 0x14 | LOAD_LOCAL | 加载局部变量 | `stack.push(locals[op[1]])` |
| 0x15 | STORE_LOCAL | 存储局部变量 | `locals[op[1]] = stack.pop()` |
| 0x16 | LOAD_GLOBAL | 加载全局变量 | `stack.push(globals[op[1]])` |
| 0x17 | STORE_GLOBAL | 存储全局变量 | `globals[op[1]] = stack.pop()` |
| 0x18 | HASH_INIT | 初始化哈希状态 | `hashState = [0x01, 0x23, ...]` |
| 0x19 | HASH_UPDATE | 更新哈希状态 | `sboxTransform()` |
| 0x1A | HASH_FINAL | 生成最终哈希 | `result = finalize()` |
| 0x1B | ENV_FEAT | 获取环境特征 | `feat = detectEnvironment()` |
| 0x1C | DBG_CHECK | 检查调试状态 | `if (elapsed > 500) dbgTriggered = true` |
| 0x1D | DBG_START | 开始调试计时 | `dbgStartTime = now()` |
| 0x1E | DBG_END | 结束调试计时 | - |
| 0x1F | SBOX_GET | 获取 S-Box 值 | `stack.push(sbox[op[1]])` |
| 0x20 | SBOX_SET | 设置 S-Box 值 | `sbox[op[1]] = stack.pop()` |
| 0x21 | B64_ENC | Base64 编码 | `stack.push(b64Encode(stack.pop()))` |
| 0x22 | B64_DEC | Base64 解码 | `stack.push(b64Decode(stack.pop()))` |
| 0x23 | HEX_ENC | 十六进制编码 | `stack.push(hexEncode(stack.pop()))` |
| 0x24 | STR_LEN | 字符串长度 | `stack.push(str.length)` |
| 0x25 | STR_CAT | 字符串拼接 | `stack.push(a + b)` |
| 0x26 | DECRYPT_INNER | 解密内层字节码 | `innerBytecode = decrypt(strPool[2])` |

### 2.4 环境检测机制

```javascript
case OP.ENV_FEAT: {
  const nav = deps[0];  // navigator
  const doc = deps[1];  // document
  const win = deps[2];  // window
  const perf = deps[3]; // performance
  
  let feat = 0;
  
  // 检测浏览器特征
  if (nav && 'userAgent' in nav) feat ^= 0x21;
  if (nav && 'language' in nav) feat ^= 0x43;
  if (doc && 'querySelectorAll' in doc) feat ^= 0x87;
  if (win && 'chrome' in win) feat ^= 0x65;
  if (perf && 'now' in perf) feat ^= 0x93;
  
  // 检测 Node.js 特征（返回错误值）
  if (typeof process !== 'undefined' && process.versions && process.versions.node) {
    feat ^= 0xDE; // 破坏特征值
  }
  
  stack.push(feat);
}
```

**关键点**：环境特征值会影响哈希状态的初始化，Node.js 环境会生成错误的特征值，导致签名失败。

### 2.5 反调试机制

```javascript
case OP.DBG_CHECK: {
  if (this.dbgStartTime > 0) {
    const elapsed = deps[4].now() - this.dbgStartTime;
    if (elapsed > 500) { // 执行超过 500ms
      this.dbgTriggered = true;
      // 破坏哈希状态
      this.hashState = this.hashState.map(b => b ^ 0xFF);
    }
  }

```

**关键点**：如果 VM 执行时间超过 500ms（可能是调试导致的），会破坏签名结果。

## 3. 签名算法核心实现

### 3.1 自定义哈希算法

签名使用 **32字节状态的非线性哈希算法**：

```javascript
// 哈希初始化
case OP.HASH_INIT: {
  this.hashState = [
    0x01, 0x23, 0x45, 0x67, 0x89, 0xAB, 0xCD, 0xEF,
    0xFE, 0xDC, 0xBA, 0x98, 0x76, 0x54, 0x32, 0x10,
    0x11, 0x22, 0x33, 0x44, 0x55, 0x66, 0x77, 0x88,
    0x99, 0xAA, 0xBB, 0xCC, 0xDD, 0xEE, 0xFF, 0x00
  ];
}

// 哈希更新（S-Box 非线性变换）
case OP.HASH_UPDATE: {
  const input = stack.pop();
  const index = stack.pop();
  
  // 每 7 字节跳跃更新状态
  const pos = (index * 7) % 32;
  
  // S-Box 非线性变换
  this.hashState[pos] = this.sbox[this.hashState[pos] ^ input];
  this.hashState[pos + 1] = this.sbox[(this.hashState[pos + 1] + input) & 0xFF];
  
  // 循环移位
  const temp = this.hashState[pos];
  for (let i = 0; i < 7; i++) {
    const nextPos = (pos + i) % 32;
    this.hashState[nextPos] = this.hashState[(nextPos + 1) % 32];
  }
  this.hashState[(pos + 7) % 32] = temp;
}

// 哈希输出
case OP.HASH_FINAL: {
  // 将 32 字节状态转换为十六进制字符串
  let result = '';
  for (let i = 0; i < 32; i++) {
    result += this.hashState[i].toString(16).padStart(2, '0');
  }
  this.result = result;
}
```

### 3.2 字节码解密算法

内层字节码使用 **XOR + S-Box 动态密钥** 加密：

```javascript
case OP.DECRYPT_INNER: {
  const encryptedHex = this.strPool[2]; // 加密的字节码
  const encryptedBytes = hexToBytes(encryptedHex);
  
  // 生成动态密钥
  const keyOffset = stack.pop();
  const keyIndices = [stack.pop(), stack.pop(), stack.pop(), stack.pop()];
  
  const key = [
    this.sbox[(keyOffset + keyIndices[0]) % 256],
    this.sbox[(keyOffset + keyIndices[1]) % 256],
    this.sbox[(keyOffset + keyIndices[2]) % 256],
    this.sbox[(keyOffset + keyIndices[3]) % 256]
  ];
  
  // XOR 解密
  const decrypted = [];
  for (let i = 0; i < encryptedBytes.length; i++) {
    decrypted[i] = encryptedBytes[i] ^ this.sbox[(i + keyOffset) % 256] ^ key[i % 4];
  }
  
  this.innerBytecode = decrypted;
}
```

### 3.3 签名生成流程

完整签名生成流程：

```javascript
function generateSignature(path, body, ts, nonce, salt) {
  // 1. 初始化 VM
  const vm = new VM(SBOX_HEX, B64_ALPHABET, OUTER_BYTECODE_HEX, STR_POOL, false);
  
  // 2. 准备输入参数
  const inputs = [path, body, ts, nonce, salt];
  
  // 3. 执行外层字节码（4字节）
  //    - DECRYPT_INNER: 解密内层字节码
  //    - EXEC_INNER: 执行内层字节码
  //    - RETURN: 返回签名结果
  
  vm.execute(vm.bytecode, 0);
  
  // 4. 内层字节码执行逻辑：
  //    a. 初始化哈希状态
  //    b. 处理路径（每个字符 -> HASH_UPDATE）
  //    c. 处理请求体（每个字符 -> HASH_UPDATE）
  //    d. 处理时间戳（每个字符 -> HASH_UPDATE）
  //    e. 处理 nonce（每个字符 -> HASH_UPDATE）
  //    f. 处理动态盐（每个字符 -> HASH_UPDATE）
  //    g. 环境特征值 -> HASH_UPDATE
  //    h. 生成最终哈希 -> HASH_FINAL
  
  return vm.result; // 64字符十六进制签名
}
```

## 4. Node.js 签名生成器

### 4.1 实现要点

Node.js 环境下的关键挑战：

1. **环境检测问题**：Node.js 没有 `window.chrome`、`document.querySelectorAll` 等特征
2. **依赖模拟**：需要模拟浏览器对象
3. **性能.now()**：需要使用 `Date.now()` 替代

### 4.2 解决方案

```javascript
// sign-generator-node.js

// 模拟浏览器环境依赖

  navigator: {
    userAgent: 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) Chrome/135.0.0.0',
    language: 'zh-CN'
  },
  document: {
    querySelectorAll: () => [],
    createElement: () => ({})
  },
  window: {
    chrome: { runtime: {} }
  },
  performance: {
    now: () => Date.now()
  }
};

// 环境特征生成
function generateEnvFeature() {
  let feat = 0;
  feat ^= 0x21; // navigator.userAgent
  feat ^= 0x43; // navigator.language
  feat ^= 0x87; // document.querySelectorAll
  feat ^= 0x65; // window.chrome
  feat ^= 0x93; // performance.now
  return feat;
}

// VM 类实现（简化版）
class VM {
  constructor() {
    this.sbox = hexToBytes(SBOX_HEX);
    this.hashState = INIT_HASH_STATE;
    this.envFeature = generateEnvFeature();
  }
  
  run(path, body, ts, nonce, salt) {
    // 直接执行签名逻辑（跳过字节码解释）
    this.initHash();
    this.processString(path);
    this.processString(body);
    this.processString(ts);
    this.processString(nonce);
    this.processString(salt);
    this.processEnvFeature();
    return this.finalizeHash();
  }
}
```

## 5. 更新检测机制

### 5.1 自动检测系统

当服务端修改加密参数时，需要同步更新签名生成器。

```javascript
// signature-updater.js

async function checkForUpdates() {
  // 1. 获取服务端最新代码
  const html = await fetch('/').then(r => r.text());
  
  // 2. 提取关键参数
  const newParams = {
    SBOX_HEX: extractSbox(html),
    B64_ALPHABET: extractB64(html),
    bytecodeHex: extractBytecode(html)
  };
  
  // 3. 读取当前参数
  const currentParams = readCurrentParams();
  
  // 4. 比较变化
  const changes = compareParams(currentParams, newParams);
  
  if (changes.length > 0) {
    console.log('⚠ 检测到参数变化，需要更新');
    return { hasChanges: true, changes };
  }
  
  return { hasChanges: false };
}
```

### 5.2 参数变化类型

| 参数 | 变化影响 | 更新难度 |
|------|---------|---------|
| S-Box | 签名结果完全不同 | 低（替换常量） |
| Base64 Alphabet | 编码结果不同 | 低（替换常量） |
| 外层字节码 | 解密密钥变化 | 中（重新解密） |
| 字符串池 | 字节码数据变化 | 中（重新提取） |
| VM 类逻辑 | 执行逻辑变化 | 高（重新分析） |

### 5.3 更新流程

```bash
# 步骤1: 检测变化
node signature-updater.js

# 步骤2: 如果检测到变化
# - S-Box/Base64 变化：直接替换 sign-generator-node.js 中的常量
# - 字节码变化：运行 decrypt-bytecode.js 解密新字节码
# - VM 类变化：重新分析 net.html

# 步骤3: 测试新签名
node api-client.js
```

## 6. 防护加强方案

### 6.1 当前系统弱点分析

| 弱点 | 风险等级 | 当前状态 | 破解难度 |
|------|---------|---------|---------|
| 静态密钥 | 🔴 高 | S-Box/Base64/字节码硬编码 | 低 |
| 可调试 | 🔴 高 | 控制台可直接调用 | 低 |
| 环境检测简单 | 🟡 中 | 只检测 navigator/document | 中 |
| 反调试弱 | 🟡 中 | 只检测执行时间 >500ms | 中 |
| 单层加密 | 🟡 中 | 字节码只做一层 XOR | 中 |
| 无设备绑定 | 🟡 中 | 签名未绑定硬件特征 | 中 |

```javascript
  }
```
```javascript
}
```

## AI Browser - 产品设计文档

版本：2.0.0 | 更新日期：2026-07-02

---

## 1. 产品概述

### 产品愿景

让每个浏览器用户都拥有一个"会思考、能操作"的 AI 助手。AI Browser 将大语言模型的理解能力与浏览器自动化操作能力深度融合，使用户通过自然语言即可完成从信息查询到复杂数据处理的各类网页任务。

### 产品定位

面向专业用户的浏览器级 AI 自动化平台。不同于通用 AI 聊天工具，AI Browser 的核心差异化在于：它不仅"能聊"，更"能做"——自主操作网页、调用脚本、追踪进度、输出结果。

### 目标用户画像

**用户 A — 信息研究员**
- 日常需要大量浏览网页、提取和整理信息
- 痛点：手动复制粘贴效率低，多页跳转操作重复
- 需求：一句话完成信息采集和整理

**用户 B — 数据分析师**
- 需要从多个网站定期采集结构化数据
- 痛点：手动采集耗时，编写爬虫门槛高
- 需求：低门槛的网页数据采集工具

**用户 C — 运营人员**
- 负责竞品监控、内容运营、定期数据上报
- 痛点：重复性监控工作占用大量时间
- 需求：定时自动化执行 + 结果导出

**用户 D — 普通浏览器用户**
- 日常浏览中需要翻译、解释、改写等辅助
- 痛点：频繁在浏览器和 AI 工具间切换
- 需求：浏览过程中即时获得 AI 帮助

### 竞品分析要点

| 维度 | AI Browser | 通用 AI 聊天工具 | 传统 RPA 工具 | 爬虫框架 |
|------|-----------|----------------|-------------|---------|
| 交互方式 | 自然语言 + 自主操作 | 纯文本对话 | 可视化流程编排 | 代码编写 |
| 网页操作能力 | 15+ DOM 工具 + 脚本 | 无 | 录制回放 | 代码实现 |
| AI 理解能力 | 页面内容理解 + 任务规划 | 强 | 弱 | 无 |
| 使用门槛 | 低（一句话描述） | 低 | 中 | 高 |
| 批量处理 | 服务端脚本支持 | 不支持 | 支持 | 支持 |
| 进度可视化 | 三阶段待办追踪 | 无 | 部分支持 | 无 |

## 2. 需求分析

### 用户痛点

1. **信息提取效率低：** 在网页间跳转、复制粘贴、整理格式，大量时间花在机械性操作上
2. **自动化门槛高：** 传统方案（爬虫、RPA）需要编程知识或复杂的流程编排
3. **AI 工具与浏览器割裂：** 需要在 AI 工具和浏览器间频繁切换，上下文丢失
4. **任务过程不透明：** 现有自动化工具执行过程黑盒，用户无法了解进展和干预
5. **重复任务无法复用：** 每次执行类似任务都要重新描述或配置

### 核心需求列表

| 需求编号 | 需求名称 | 优先级 | 描述 | 对应功能 |
|---------|---------|--------|------|---------|
| R01 | 网页内容问答 | P0 | 基于当前页面内容与 AI 对话 | AI 智能对话 |
| R02 | 自主网页操作 | P0 | AI 自主规划并执行网页操作任务 | Agent 智能助手 |
| R03 | 文本处理工具 | P1 | 选中文字后快速获取翻译/解释/改写/摘要 | 划词工具 |
| R04 | 表单自动填充 | P1 | AI 分析表单并自动填充合理值 | 智能表单填充 |
| R05 | 脚本执行管理 | P1 | 管理和执行服务端脚本完成高级操作 | 脚本管理 |
| R06 | 定时自动化 | P2 | 周期性执行网页操作任务 | 定时任务 |
| R07 | 网络请求分析 | P2 | 捕获和分析页面网络请求 | 网络请求捕获 |
| R08 | 任务模板复用 | P2 | 保存和快速启动常用任务 | 任务模板 |
| R09 | 执行过程录制 | P2 | 记录工具调用过程供复盘 | 工具录制 |
| R10 | 多格式数据导出 | P1 | 将结果导出为 JSON/CSV/MD/HTML/TXT | 数据导出 |
| R11 | 进度可视化追踪 | P0 | 实时展示 Agent 执行进度和状态 | 待办追踪面板 |
| R12 | 文件上传分析 | P1 | 上传图片/PDF 供 AI 分析 | 文件上传 |

### 非功能需求

- **性能：** AI 对话流式输出，Agent 步骤实时反馈，无明显卡顿
- **安全：** 请求签名、域名策略、XSS 防护、URL 检查、Nonce 验证
- **兼容性：** 支持 Chrome 及基于 Chromium 的浏览器（Edge 等）
- **可靠性：** Port 断开不终止 Agent，支持断点恢复和重连
- **可扩展性：** 服务模块化设计，支持快速添加新工具和面板

### 导航结构

产品采用扁平化导航，核心入口集中在 SidePanel 的顶部导航栏：

- **对话** | **Agent** | **脚本** | **设置** — 四个一级入口
- **返回按钮** — 从子视图返回对话
- **功能面板** — 通过浮动按钮打开，独立于主导航

Popup 作为快捷入口，所有操作最终引导至 SidePanel。

## 4. 用户流程设计

### 4.1 AI 对话流程

```
用户打开侧边栏
  │
  ├── 输入文字消息
  │     │
  │     ├── 检测关键词（总结/翻译/提取）
  │     │     ├── 命中 → 自动请求页面内容 → 注入消息上下文
  │     │     └── 未命中 → 直接发送
  │     │
  │     ▼
  │   Background 调用 AI API（流式）
  │     │
  │     ▼
  │   SidePanel 逐字符渲染回复（Markdown）
  │     │
  │     ▼
  │   用户可追问 / 新建对话
  │
  ├── 上传图片
  │     └── 转 base64 → 附加到消息 → 需视觉模型
  │
  └── 上传 PDF
        └── 发送到服务端解析 → 提取文本 → 附加到消息
```

**异常处理：**
- 网络错误：显示错误提示，保留输入内容
- 扩展上下文失效：提示用户关闭并重新打开侧边栏
- API 超时：显示超时提示，可重试

### 4.2 Agent 任务执行流程

```
用户切换到 Agent 模式 → 输入任务描述 → 发送
  │
  ▼
系统自动准备
  ├── 读取当前页面内容
  ├── 提取关键词搜索服务端工具库
  └── 注入页面上下文到 AI 提示词
  │
  ▼
AI 规划任务
  ├── AI 创建三阶段待办列表
  ├── 系统校验合规性（工具权限 / 数据依赖 / 唯一性）
  │     ├── 校验通过 → 进入执行
  │     └── 校验失败 → 返回错误，AI 修正后重试
  │
  ▼
Stage 1 页面探索
  ├── AI 调用 DOM 工具操作页面
  ├── 系统匹配工具调用到当前待办 → 更新进度
  ├── 待办面板实时刷新
  │     │
  │     ├── 正常完成 → 切换 Stage 2
  │     └── 连续 4 次无进展 → 强制切换 Stage 2
  │
  ▼
Stage 2 脚本处理
  ├── AI 调用 inject_script_N 执行脚本
  ├── 脚本结果存入全局数据_store
  │     │
  │     ├── 正常完成 → 切换 Stage 3
  │     └── 连续 3 次脚本失败 → 强制切换 Stage 3
  │
  ▼
Stage 3 结果汇总
  ├── AI 查询已存储数据
  ├── AI 调用 finish_task 输出汇总
  ├── 流式展示结果
  │
  ▼
事后评判（可选）
  ├── AI 评判任务完成度
  └── 追加评估结果到输出
```

**进度展示与反馈设计：**
- 每个工具调用生成一张步骤卡片：工具名、参数、执行状态（等待/运行/完成/失败）、结果摘要
- 页面内待办面板同步更新：阶段指示器、进度条、待办列表状态、最近工具调用
- 收敛提示以系统消息形式注入，引导 AI 加快推进

**异常与中断处理：**
- 超时：显示超时提示，建议简化任务
- 工具调用次数超限：显示超限提示
- API 错误：重试 2 次后显示错误详情
- Port 断开：Agent 继续运行，SidePanel 重连后恢复
- 导航操作后页面跳转：自动终止当前序列的后续工具调用

**人机交互设计：**
- Agent 运行中遇到需要用户决策的情况 → 发送介入请求到 SidePanel
- 用户在 SidePanel 中响应 → Agent 获取答案继续执行

### 4.3 划词工具流程

```
用户在网页上选中文本（≥3 字符）
  │
  ▼
mouseup 事件 → 100ms 延迟后检测选区
  │
  ├── 有选区 → 计算位置 → 弹出工具栏（Shadow DOM）
  │     │
  │     ├── 点击操作按钮（解释/翻译/改写/摘要）
  │     │     └── 存储 pendingMessage → 请求打开 SidePanel
  │     │           └── SidePanel 读取消息 → 发送到 AI → 展示结果
  │     │
  │     └── 点击其他区域 → 移除工具栏
  │
  └── 无选区 → 不触发
```

### 4.4 智能表单填充流程

```
页面加载 → MutationObserver 扫描表单
  │
  ├── 检测到表单（≥2 个可填字段，非纯搜索框）
  │     └── 在提交按钮旁注入"智能填充"按钮
  │
  ▼
用户点击"智能填充"
  │
  ├── 分析所有字段（标签、类型、占位符、选项、必填）
  ├── 发送 formFillRequest 到 Background
  │     └── Background 调用 AI 生成填充值（JSON 格式）
  │
  ▼
收到 AI 响应 → 逐字段填入
  ├── input/textarea: 使用原生 setter 赋值 + 触发 input/change 事件
  ├── select: 匹配选项文本
  ├── radio: 匹配 value 值
  └── checkbox: 解析布尔值
  │
  ▼
表单短暂高亮（绿色光晕 2 秒）→ 用户检查并修改 → 提交
```

### 4.5 脚本管理流程

```
安装/定时/手动触发同步
  │
  ▼
从 Admin Server 拉取脚本列表
  └── 保存到 chrome.storage.local
  │
  ▼
用户在 Popup/SidePanel 查看脚本列表
  ├── 启用/禁用脚本
  └── 点击执行
        │
        ▼
      ScriptService 注入脚本到当前页面
        ├── precheck 前置校验
        ├── 执行脚本代码
        └── 回调结果 → 存储到 injectCallbacks
```

### 4.6 定时任务流程

```
用户在功能面板创建定时任务
  ├── 选择操作类型（导航 / 脚本 / Agent 消息）
  ├── 配置参数（URL / 脚本 ID / 消息内容）
  ├── 设置调度方式（Cron 表达式 / 固定间隔）
  └── 保存到 IndexedDB
  │
  ▼
chrome.alarms 每分钟触发心跳
  │
  ▼
ScheduledTaskService 检查到期任务
  ├── 导航：chrome.tabs.update 打开 URL
  ├── 脚本注入：ScriptService 执行脚本
  └── Agent 消息：发送 scheduledAgentMessage 事件
```

## 5. 界面设计说明

### 5.1 Popup 弹窗

**布局：** 固定宽度的弹出窗口，顶部 Tab 栏切换三个面板。

| Tab | 功能分布 |
|-----|---------|
| AI 助手 | 输入框 + 快捷按钮（总结/翻译/提取），所有操作引导至 SidePanel |
| 脚本 | 脚本列表（名称/描述/开关）+ 同步按钮 |
| 设置 | 服务器地址、AppKey/Secret、模型下拉、划词工具开关 |

**与 SidePanel 的关系：** Popup 是轻量入口，通过 `pendingMessage` 传递消息到 SidePanel。复杂的对话和 Agent 操作在 SidePanel 中完成。

### 5.2 SidePanel 侧边栏

**整体布局：**

```
┌──────────────────────────┐
│  导航栏: 对话 | Agent | 脚本 | 设置   │
├──────────────────────────┤
│                           │
│      主内容区域             │
│  （聊天 / Agent 步骤 / 脚本列表 / 设置） │
│                           │
│                           │
├──────────────────────────┤
│  输入区: [附件] [输入框] [发送]        │
│        [总结页面] [工具箱]             │
└──────────────────────────┘
```

**聊天模式界面：**
- 消息气泡布局（用户消息右对齐，AI 回复左对齐）
- AI 回复支持 Markdown 渲染（标题、列表、代码块、链接）
- 流式逐字符输出动画
- 消息区域可滚动，自动滚到底部
- 顶部浮动"新建对话"按钮

**Agent 模式界面：**
- 输入框发送任务指令
- 步骤卡片流：每个工具调用一张卡片
  - 卡片头部：步骤序号、工具名称、状态图标
  - 卡片内容：工具参数、执行结果（可折叠）
  - 特殊卡片：搜索结果列表、finish_task 汇总
- 调试日志按钮（打开功能面板）

**设置面板：**
- 表单布局，分组展示配置项
- 服务器配置组：地址、AppKey、AppSecret
- AI 配置组：模型选择器（按供应商分组，标注视觉/工具能力标签）
- Agent 配置组：最大轮次、评判开关、调试模式
- 功能开关组：划词工具

**功能面板（浮动窗口）：**
- 可拖动、可最小化的浮动窗口
- 标签页切换：执行图谱、资源监控、任务模板、工具录制、定时任务、调试日志
- 拖动功能通过 mousedown/mousemove/mouseup 实现

### 5.3 页面内元素

**浮动操作按钮（右侧边缘）：**
- 5 个按钮垂直排列，固定在页面右下角上方
- 药丸形（pill）和图标形（icon）两种样式
- AI 按钮使用品牌渐变色突出显示
- 悬停展开显示标签文字
- 点击时短暂缩放动画反馈

**划词工具栏：**
- 出现在选中文本上方，居中对齐
- 白色背景，圆角卡片，轻微阴影
- 四个按钮等距排列
- 入场动画：从下方淡入上移

**智能表单填充按钮：**
- 内联在表单提交按钮旁
- 紫色主题，带机器人图标
- 执行中显示旋转加载图标
- 完成后表单高亮闪烁

**待办追踪面板：**
- 固定在页面右上方（可拖动调整位置）
- 头部：标题 + 复制/清空/关闭按钮（可拖动）
- 概览栏：当前阶段、进度条、完成数、最近工具调用
- 待办列表：按阶段分组，每项显示状态图标、ID、工具名、描述、数据键
- 阶段切换日志：黄色提示条

### 5.4 右键菜单

三个菜单项，由 Chrome 扩展 API 注册：
- 页面上下文：「AI 总结此页面」「翻译此页面」
- 选区上下文：「AI 解释选中文字」

### 5.5 Todo 查看器窗口

独立的 Chrome 窗口，用于在全屏模式下查看待办进度。与页面内嵌面板共享数据源，通过 `chrome.windows.create` 打开。

## 6. 交互设计规范

### 反馈机制

| 操作 | 加载状态 | 成功反馈 | 失败反馈 |
|------|---------|---------|---------|
| AI 对话 | 逐字符流式输出 | 完整渲染回复 | 错误提示卡片 |
| Agent 启动 | "思考中..." 状态 | agentStart 事件 | 错误提示 |
| 工具调用 | 步骤卡片 loading | 结果展示 + 进度更新 | 错误详情 |
| 表单填充 | 按钮旋转图标 | 表单绿色高亮 2s | 右上角红色提示 3s |
| 脚本同步 | 按钮禁用 | 列表刷新 | Toast 提示 |
| 划词操作 | 打开 SidePanel | AI 回复展示 | 错误提示 |

### 进度展示设计

**步骤卡片流：**
- 每张卡片包含：序号、工具名称标签、参数 JSON（可折叠）、结果文本
- 状态色：蓝色（运行中）、绿色（成功）、红色（失败）、黄色（等待重试）
- 搜索结果以列表卡片形式展示

**待办面板：**
- 三阶段进度条 + 百分比
- 每个待办项的状态图标：⬜（待处理）、⏳（执行中）、✅（完成）、❌（失败）
- 阶段头部颜色区分：蓝色（Stage 1）、黄色（Stage 2）、绿色（Stage 3）
- 阶段切换日志：黄色带图标的提示条

**执行图谱：**
- 三阶段流水线可视化
- 当前阶段高亮
- 阶段内工具调用连线

### 错误处理与用户引导

- **扩展上下文失效：** 提示"扩展已更新，请刷新页面后使用"
- **API 超时：** 显示"API 响应较慢，请耐心等待" → 超时后建议重试
- **Agent 超时：** "Agent 执行超时，请简化任务后重试"
- **工具次数超限：** "工具调用次数超限，请简化任务重试"
- **域名拦截：** "导航被安全策略阻止：[URL] 不在允许的域名范围内"
- **网络错误：** 红色提示卡片，保留用户输入内容

### 动画与过渡效果

- 划词工具栏入场：`toolbarIn` 动画（0.2s，从下方淡入上移）
- 待办面板入场：`todoIn` 动画（0.25s，从右侧淡入）
- 按钮点击：`scale(0.92)` 缩放 + 150ms 恢复
- 表单填充完成：`box-shadow` 绿色光晕 2s 渐隐
- 浮动按钮悬停：背景色渐变 + 标签淡入

## 7. 功能规格说明

### 7.1 AI 对话功能规格

**输入：**
- 文本消息（无长度限制，服务端截断）
- 图片（base64 Data URL，需视觉模型）
- PDF 文件（服务端解析提取文本）

**输出：**
- Markdown 渲染（标题、列表、代码块、链接、粗体/斜体）
- 代码块支持语言标识
- 链接安全过滤（仅 http/https/mailto）

**历史记录管理：**
- 自动保存到 `chrome.storage.local`
- 字符级截断（约 8000 字符）
- 最多 50 条消息
- 新建对话清空当前上下文

**模型选择：**
- 从服务端获取可用模型列表
- 按供应商分组显示
- 标注能力标签：视觉（vision）、工具调用（tools）
- 当前默认：deepseek-v4-pro

### 7.2 Agent 功能规格

**DOM 操作工具清单：**

| 工具名 | 功能 | 参数 |
|--------|------|------|
| read_page_content | 读取页面标题/URL/正文 | - |
| extract_content | CSS 选择器提取内容 | selector, multiple, limit, attributes |
| click_element | 点击元素 | selector, index |
| fill_input | 填写输入框 | selector, value, submit |
| wait_for_element | 等待元素出现 | selector, timeout |
| navigate_to | 导航到 URL | url |
| go_back / go_forward | 浏览器前进/后退 | - |
| scroll_page | 滚动页面 | direction, amount |
| hover_element | 悬停元素 | selector, index |
| select_dropdown | 选择下拉框 | selector, value, by |
| press_key | 模拟按键 | key, selector |
| find_text_on_page | 页面文本搜索 | query, caseSensitive |
| get_element_info | 元素详细信息 | selector, limit, attributes |
| get_interactive_elements | 可交互元素列表 | selectorHint |
| screenshot_visible | 截取可见区域 | - |

**任务规划规则：**
- 三阶段待办列表由 AI 创建，系统校验
- Stage 1 只能用 DOM 工具
- Stage 2 只能用 `inject_script_N` 和辅助工具
- Stage 3 只能用 `recall_data` 和 `finish_task`
- dataOutputKey 必须唯一，dataDependKeys 必须引用已存在的 key
- 待办创建后不可修改

**收敛策略：**
- 70% 预算 → 软警告："请加快推进核心待办"
- 85% 预算 → 紧急收敛："请立即完成或 finish_task"
- Stage 1 连续 4 次无进展 → 自动切换 Stage 2
- Stage 2 连续 3 次脚本失败 → 自动切换 Stage 3
- 最大轮次（默认 15，可配置）→ 终止并提示

**评判机制：**
- 可选启用 AI 评判器
- 评估维度：任务完成度（success / partial / failure）
- 评判结果追加到输出中，附带评语

### 7.3 划词工具功能规格

### 7.4 脚本管理功能规格

- 记录维度：成功/失败、执行耗时、错误信息、结果摘要
- 影响脚本排序：成功率高的脚本优先推荐
- 跨用户共享：服务端聚合所有用户的执行经验

## 10. 设计决策记录

| 决策 | 选择方案 | 理由 | 替代方案 |
|------|---------|------|---------|
| 无构建步骤 | 纯 Vanilla JS + ES Modules | 降低开发门槛，快速迭代，Chrome 原生支持 | Webpack/Vite 构建 |
| Shadow DOM 隔离 | `mode: 'closed'` | 完全隔离页面样式和脚本干扰 | `mode: 'open'` + CSS 命名空间 |
| 三阶段 Agent 模型 | DOM → 脚本 → 汇总 | 渐进式能力释放，先探索再批量处理 | 单阶段混合工具 |
| 待办调度系统 | 系统驱动进度（非 AI 自主判断） | 客观追踪，避免 AI 自我评估偏差 | AI 自主管理进度 |
| Port 弹性设计 | 断开不终止 Agent | 用户可能意外关闭侧边栏，任务不应中断 | 断开即终止 |
| 上下文压缩 | 四级分层压缩 | 保留关键信息，大幅降低 Token 消耗 | 固定窗口截断 |
| Nonce 回调验证 | `enumerable: false` 随机 Nonce | 防止页面脚本枚举和伪造 | 无验证 / 固定密钥 |
| 附件持久化 | localStorage（非 chrome.storage） | 同步 API，避免异步时序和 SW 冷启动问题 | chrome.storage.local |
| HMAC-SHA256 纯 JS 实现 | 自实现（非 crypto.subtle） | 兼容 HTTP 页面上下文（非安全上下文无 crypto） | Web Crypto API |
| 浮动按钮 MutationObserver 守护 | 移除后自动重注入 | 某些页面脚本会清理"异常" DOM 元素 | 一次性注入不守护 |

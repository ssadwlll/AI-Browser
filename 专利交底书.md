# 专利交底书

## 发明名称

**一种基于大语言模型自主决策的浏览器智能操控方法及系统**

---

## 一、技术领域

本发明涉及浏览器自动化控制技术领域，具体涉及一种结合大语言模型（LLM）Function Calling 机制与多轮工具调用闭环的浏览器智能操控方法及系统。

---

## 二、背景技术（现有技术的问题）

### 2.1 现有浏览器自动化方案的局限

目前主流的浏览器自动化方案包括：

| 方案 | 实现方式 | 局限 |
|------|---------|------|
| Selenium / Puppeteer | 编写脚本调用 WebDriver API | 需要编程能力，无法应对非预期页面变化 |
| RPA 工具 | 录制操作步骤回放 | 依赖固定选择器，页面结构变化即失效 |
| 浏览器扩展 | 注入脚本执行特定功能 | 功能单一，无法组合和决策 |

上述方案的共同缺陷是：**预设规则驱动，缺乏自主理解和决策能力**。用户必须明确描述每一步操作，系统无法根据页面实际状态自适应调整策略。

### 2.2 LLM 自动化方案的不足

已有方案尝试将 LLM 与浏览器结合，但存在以下问题：

1. **单一工具调用**：LLM 只能调用一两个预设工具，无法覆盖复杂场景
2. **缺乏闭环反馈**：执行结果不反馈给 LLM，无法形成"感知→决策→执行→反馈"的闭环
3. **无页面导航感知**：代码执行导致页面跳转后，上下文丢失，后续操作失败
4. **无错误自愈**：执行失败后无法自动分析和重试

---

## 三、发明内容

### 3.1 要解决的技术问题

本发明要解决的技术问题是：**如何让大语言模型像人类一样自主操作浏览器，持续感知页面状态、自主决策工具调用、感知执行结果并迭代调整，直到完成用户给定的复杂任务。**

### 3.2 技术方案核心流程

```
┌─────────────────────────────────────────────────────────────────┐
│                      用户输入自然语言任务                         │
│              "提取本页所有商品的价格和名称，按价格排序"              │
└────────────────────────────┬────────────────────────────────────┘
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│  第1轮：AI 分析任务 → 决定调用 collect_page_context 了解页面      │
│  → 客户端执行工具 → 返回 DOM 结构摘要给 AI                        │
└────────────────────────────┬────────────────────────────────────┘
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│  第2轮：AI 分析 DOM → 决定调用 execute_js 提取数据                │
│  → 客户端注入 JS 代码 → 返回提取结果                              │
└────────────────────────────┬────────────────────────────────────┘
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│  第3轮：AI 检查结果 → 数据完整 → 返回最终文本回复（不调用工具）     │
│  → 闭环结束，任务完成                                             │
└─────────────────────────────────────────────────────────────────┘
```

### 3.3 核心技术方案

#### 3.3.1 多工具定义体系（14 种浏览器操作工具）

基于 OpenAI Function Calling 协议，定义了 14 种浏览器操作工具，每种工具包含名称、自然语言描述、结构化参数 schema：

| 工具名称 | 功能 | 参数 |
|---------|------|------|
| `collect_page_context` | 收集页面上下文（URL、标题、DOM 结构摘要） | max_elements |
| `execute_js` | 在页面中执行任意 JavaScript 代码 | code, description |
| `get_network_requests` | 获取页面捕获的网络请求数据 | filter_url, method, limit |
| `navigate_to` | 导航到指定 URL | url |
| `extract_page_scripts` | 提取页面 JS 脚本信息 | include_inline, max_content_length |
| `get_page_html` | 获取页面 HTML 源代码 | selector, max_length |
| `screenshot` | 对当前页面截图 | — |
| `click_element` | 点击指定元素 | selector, index, wait_after_click |
| `wait_for_element` | 等待元素出现 | selector, timeout, visible |
| `wait_for_navigation` | 等待页面导航完成 | timeout |
| `open_new_tab` | 在新标签页打开 URL | url, active |
| `close_current_tab` | 关闭当前标签页 | — |
| `extract_images` | 提取页面图片信息 | min_width, min_height, limit |
| `extract_links` | 提取页面链接信息 | domain_only, filter, limit |
| `scroll_to_element` | 滚动到指定元素 | selector, behavior, offset |

**创新点**：AI 根据任务需求**自主决定**调用哪些工具、以什么顺序调用、传什么参数，无需人工预设工作流。

#### 3.3.2 多轮工具调用闭环机制

核心流程伪代码：

```
function unifiedChatLoop(messages, maxRounds):
  for round = 1 to maxRounds:
    // 步骤1: 将对话上下文+工具定义发送给AI
    aiMessage = llmProvider.chat(messages, tools=TOOL_DEFINITIONS)

    // 步骤2: 判断AI是否决定调用工具
    if aiMessage.tool_calls is empty:
      // AI直接返回文本回复，任务完成
      return aiMessage.content

    // 步骤3: 执行AI请求的每个工具
    for toolCall in aiMessage.tool_calls:
      toolResult = toolExecutor.execute(
        toolCall.function.name,
        toolCall.function.arguments
      )

    // 步骤4: 将工具执行结果反馈给AI
    messages.push(aiMessage)
    messages.push({ role: 'tool', content: toolResult })

    // 步骤5: 继续下一轮循环
```

**关键创新**：形成"**AI 决策 → 客户端执行 → 结果反馈 → AI 再决策**"的闭环，AI 可自主进行多轮迭代直到任务完成。

#### 3.3.3 页面导航感知机制

当 AI 生成的 JS 代码执行后导致页面跳转（如点击链接、提交表单），系统自动检测并处理：

```
async function executeInPage(browserView, jsCode):
  urlBefore = browserView.getURL()        // 记录执行前URL

  executeJavaScript(jsCode)               // 执行AI生成的代码

  urlAfter = browserView.getURL()         // 检查执行后URL

  if urlBefore != urlAfter OR browserView.isLoading():
    // 页面发生了导航！
    waitForPageLoad(browserView)          // 等待新页面加载
    newContext = collectPageContext()     // 收集新页面DOM结构
    return { navigated: true, newContext }  // 告知AI新页面信息

  return normalResult                     // 未导航，返回正常结果
```

**创新点**：AI 无需预先知道操作会触发导航，系统自动感知导航事件并重建页面上下文，AI 可基于新页面结构继续任务。

#### 3.3.4 多模型自适应适配

系统支持多种 LLM 后端，统一接口适配：

| 模型 | 适配方式 |
|------|---------|
| OpenAI 兼容 API | 原生 Function Calling 支持 |
| Ollama 本地模型 | 原生 Function Calling 支持 |
| 通义千问 DashScope | 原生 Function Calling 支持 |
| 其他模型 | 降级方案：将工具定义转为 prompt 文本，让 AI 按格式输出 JSON |

**创新点**：自适应检测模型能力，对不支持 Function Calling 的模型自动降级为 prompt 模拟，无缝切换。

#### 3.3.5 智能体自主循环（Agent Loop）

智能体模式实现了更高级的自主决策：

```
function agentLoop(browserView, task, llmProvider):
  pageContext = collectPageContext()           // 收集初始页面状态
  messages = [systemPrompt, task + pageContext]

  while round < maxRounds and not completed:
    // 阶段1: AI流式分析，前端实时展示思考过程
    aiReply = llmProvider.chatStream(messages)

    // 阶段2: 检查任务完成标记
    if "[TASK_COMPLETE]" in aiReply:
      return aiReply.summary

    // 阶段3: 提取AI生成的JS代码
    jsCode = extractJsCode(aiReply)

    // 阶段4: 注入页面执行
    result = executeInPage(browserView, jsCode)

    // 阶段5: 检测导航并重建上下文
    if result.navigated:
      waitForPageLoad()
      newContext = collectPageContext()
      contextUpdate = formatNewPageInfo(newContext)

    // 阶段6: 将执行结果反馈给AI
    messages.push({ role: 'assistant', content: aiReply })
    messages.push({
      role: 'user',
      content: `执行结果: ${result}${contextUpdate}\n请继续完成任务。`
    })

    round++
```

**创新点**：
- AI 自主感知页面状态变化，包括导航事件
- 执行结果完整反馈，AI 可基于最新页面状态调整策略
- 支持错误自愈：执行失败时 AI 分析错误原因并重新生成代码
- 上下文窗口管理：自动裁剪历史消息防止超出 token 限制

## 四、技术效果

### 4.1 与现有技术的对比优势

| 对比维度 | 传统RPA | Selenium脚本 | 本发明方案 |
|---------|---------|-------------|-----------|
| 操作方式 | 录制回放 | 编写代码 | 自然语言描述 |
| 页面变化适应 | 需重新录制 | 需修改代码 | AI 自主调整 |
| 多步骤复杂任务 | 预设流程 | 预设流程 | AI 自主规划 |
| 错误处理 | 预设规则 | 预设 try-catch | AI 分析原因→重试 |
| 页面导航后继续 | 不支持 | 需手动处理 | 自动感知→重建上下文 |
| 跨页面操作 | 不支持 | 复杂 | AI 自主创建标签页 |

### 4.2 典型应用效果

- **数据采集**：用户说"提取所有商品价格"，AI 自主完成 DOM 分析→代码生成→执行→数据整理，无需用户知道页面结构
- **跨页面操作**：AI 在页面 A 提取链接→打开新标签页→在页面 B 继续操作，全程自主
- **错误自愈**：首次执行因选择器变化失败，AI 分析 DOM 结构后重新生成正确的选择器，再次执行成功

## 六、权利要求

### 独立权利要求 1（方法）

一种基于大语言模型自主决策的浏览器智能操控方法，其特征在于，包括以下步骤：

**步骤 S1：** 接收用户输入的自然语言任务描述，将所述任务描述与预设的系统提示、浏览器操作工具定义集合组合为初始对话上下文；

**步骤 S2：** 将所述对话上下文发送至大语言模型，大语言模型根据任务需求自主决策是否调用工具；

**步骤 S3：** 若大语言模型返回工具调用请求，则解析所述工具调用请求中的工具名称和参数，由客户端在浏览器页面中执行对应工具操作，获取执行结果；

**步骤 S4：** 将所述工具调用请求和所述执行结果追加至对话上下文，返回步骤 S2 进行下一轮决策；

**步骤 S5：** 若大语言模型返回文本回复且不包含工具调用请求，则判定任务完成，输出所述文本回复作为任务结果。

### 独立权利要求 2（系统）

一种基于大语言模型自主决策的浏览器智能操控系统，其特征在于，包括：

**工具定义模块**，用于定义多个浏览器操作工具，每个工具包含名称、自然语言描述和结构化参数 schema；

**工具执行模块**，用于接收大语言模型的工具调用请求，在浏览器页面中执行对应操作并返回执行结果；

**多轮调用闭环模块**，用于将用户任务、工具定义、AI 回复和工具执行结果组织为对话上下文，循环发送至大语言模型，直至大语言模型返回不包含工具调用的文本回复；

**页面导航感知模块**，用于在工具执行前后检测页面 URL 是否发生变化，若发生变化则等待新页面加载完成后重新收集页面上下文，并将新页面信息反馈至对话上下文。

### 从属权利要求

**权利要求 3：** 根据权利要求 1 所述的方法，其特征在于，所述浏览器操作工具包括页面上下文收集工具、JavaScript 代码注入工具、网络请求获取工具、页面导航工具、元素点击工具、元素等待工具、页面导航等待工具、页面截图工具、图片提取工具、链接提取工具、元素滚动工具中的至少一种。

**权利要求 4：** 根据权利要求 1 所述的方法，其特征在于，步骤 S3 中还包括：在执行 JavaScript 代码工具后，检测浏览器页面 URL 是否发生变化，若发生变化则等待新页面加载完成，重新收集新页面的 DOM 结构摘要，将所述新页面的 DOM 结构摘要作为上下文信息反馈至大语言模型。

**权利要求 5：** 根据权利要求 1 所述的方法，其特征在于，还包括多模型自适应步骤：检测大语言模型是否支持原生 Function Calling 协议，若支持则直接使用，若不支持则将工具定义转换为文本提示词，指导大语言模型按指定 JSON 格式输出工具调用。

**权利要求 6：** 根据权利要求 1 所述的方法，其特征在于，还包括错误自愈步骤：当工具执行失败时，将失败原因作为上下文信息反馈至大语言模型，大语言模型分析失败原因后重新生成操作指令。

**权利要求 7：** 根据权利要求 1 所述的方法，其特征在于，还包括智能体自主循环步骤：大语言模型在每轮决策中生成 JavaScript 代码片段，客户端执行后检测执行结果，若代码执行导致页面导航，则自动等待新页面加载并重建页面上下文，大语言模型基于新页面结构继续下一轮决策，直至任务完成。

**权利要求 8：** 根据权利要求 1 所述的方法，其特征在于，还包括上下文窗口管理步骤：当对话上下文消息数量超过预设阈值时，保留系统提示消息和最近的对话消息，裁剪中间历史消息，防止超出大语言模型的上下文长度限制。

**权利要求 9：** 根据权利要求 1 所述的方法，其特征在于，还包括流式输出步骤：大语言模型的文本回复和工具调用请求通过流式传输实时推送至用户界面，用户界面实时展示 AI 的思考过程和工具调用状态。

**权利要求 10：** 根据权利要求 1 所述的方法，其特征在于，所述浏览器操作工具定义集合中的每个工具均包含触发条件描述，大语言模型根据所述触发条件描述以及当前页面上下文自主判断是否满足调用条件。

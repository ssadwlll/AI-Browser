> **分析周期**：2026-07-09 ~ 2026-07-10
> **目标**：逆向分析小红书搜索接口签名机制，实现批量数据采集
> **核心结论**：
> - ✅ **搜索批量采集完全可用**：v2 API + 全部浏览器请求头 + XYS_ 静态签名复用，20关键词×3页=1300+条笔记全部成功

| 防御层 | 机制 | 破解状态 | 说明 |

## 二、签名机制解析（已破解部分）

### 2.1 mnsv2 脚本结构

mnsv2 由两个脚本组成，可从 CDN 获取后在 Node.js 中独立运行：

| 脚本 | 大小 | URL | 作用 |
|------|------|-----|------|
| ds 脚本 | ~60KB | `https://as.xiaohongshu.com/api/sec/v1/ds?appId=xhs-pc-web` | 创建运行时 + 暴露 mnsv2 函数 |
| VM 脚本 | ~246KB | `https://fe-static.xhscdn.com/as/v1/3e44/public/a9ef723c54cfdb63556bffe75cf06ae7.js` | 补充字节码数据表 + 扩展运行时 |

**ds 脚本结构**：
- `getdss()` — 返回时间戳字符串（每次请求脚本时生成，嵌入脚本内容）
- `_0x341b` — 混淆字符串数组（183项）
- `_0x1769 / _0x12372b` — 字符串解密函数
- `__$c` — 字节码字符串（~8KB 十六进制编码）
- 启动逻辑：`glb['_BHjFmfUMEtxhI'](__$c, [,, undefined, Uint8Array, getdss])`
- 内置 Node.js 兼容：`var glb = typeof window === 'undefined' ? global : window;`

**VM 脚本结构**：
- 自执行 IIFE，包含 Wm 混淆字符串数组、字节码数据表
- 不直接引用浏览器 API，仅通过闭包访问 ds 脚本建立的运行时

### 2.2 mnsv2 函数

| 属性 | 值 |
|------|-----|
| 调用方式 | `mnsv2(apiPath)` — **单参数**（API 路径字符串） |
| 输入示例 | `/api/sns/web/v1/search/notes?keyword=test` |
| 输出 | `mns0301_` + 176 字符 base64 = **固定 200 字符** |
| 幂等性 | 不幂等（相同输入每次返回不同值，含时间/随机因子） |
| 内部结构 | `mnsv2['ΙIΙ']` 字节码解释器 + `$0` 26项依赖数组 + `$1` 134项字节码 + `$2` IO映射表 |
| Node.js 兼容 | ✅ 原生支持（内置 `typeof window === 'undefined' ? global : window`） |

### 2.3 X-s 签名组装

```
mnsv2(apiPath) → mns0301_xxx (200字符)
  → 构建 Payload JSON { x0, x1, x2, x3, x4 }
  → UTF8 编码 → 标准 Base64 → 自定义 Base64 映射
  → "XYS_" 前缀
  → 最终 X-s 值
```

**Payload 结构**：
```json
{
  "x0": "4.3.7",           // 指纹版本
  "x1": "xhs-pc-web",      // 平台标识
  "x2": "PC",              // 设备类型
  "x3": "mns0301_abc...",  // mnsv2 签名结果（200字符）
  "x4": "string"            // body 参数类型
}
```

### 2.4 自定义 Base64 字母表

```
自定义: ZmserbBoHQtNP+wOcza/LpngG8yJq42KWYj0DSfdikx3VT16IlUAFM97hECvuRX5
标准:   ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/
```

### 2.6 _webmsxyw 与 XYW_ 格式（✅ 最新突破）

通过分析浏览器控制台输出和 signSvn 配置，成功在 Node.js 中运行 `_webmsxyw`：

**XYW_ 格式结构**（标准 Base64 编码的 JSON）：
```
XYW_ + Base64({
  "signSvn": "56",
  "signType": "x2",
  "appId": "xhs-pc-web",
  "signVersion": "1",
  "payload": "<416字符 hex编码, 208字节>"
})
```

**与 XYS_ 格式的对比**：

| 维度 | XYS_ 格式（旧） | XYW_ 格式（新） |
|------|-----------------|-----------------|
| 生成函数 | `mnsv2(apiPath)` | `_webmsxyw(apiPath, body)` |
| 编码方式 | 自定义 Base64 字母表 | 标准 Base64 |
| JSON 结构 | `{x0,x1,x2,x3,x4}` | `{signSvn,signType,appId,signVersion,payload}` |
| 路径绑定 | ✅ 是 | ✅ 是 |
| 请求体绑定 | ❌ 不绑定 | ✅ **绑定**（不同 body 产生不同签名） |
| 幂等性 | 非幂等 | 非幂等（前192字节确定，后16字节随机） |
| Node.js 运行 | ❌ vm.js 加载失败 | ✅ **可运行**（ds.js + sign.js） |
| X-s-common | 需要单独生成 | 不返回（需另行测试是否必需） |

**脚本依赖关系**：

| 脚本 | 来源 | 作用 | 大小 |
|------|------|------|------|
| ds.js | `https://as.xiaohongshu.com/api/sec/v1/ds?appId=xhs-pc-web` | 创建运行时（`_BHjFmfUMEtxhI`） | ~60KB |
| sign.js | signSvn 配置中的 `signUrl` | 定义 `_webmsxyw` 函数 | ~148KB |
| xhsTokenUrl | signSvn 配置中的 `xhsTokenUrl` | 标记脚本（仅45字节，`_as_m1125=1`） | 45B |

**signSvn 配置**（`localStorage.sdt_source_storage_key`）：

## 三、实测验证结果（重大发现）

### 3.1 字段必需性测试

通过控制变量法测试各参数的必需性：

| 测试 | x-s | x-s-common | filters | 结果 | 结论 |
|------|-----|------------|---------|------|------|
| 完整请求 | ✅ | ✅ | ✅ | ✅ 22条 | 基线成功 |
| 无 x-s-common | ✅ | ❌ | ✅ | ❌ 300011 | **x-s-common 必需** |
| 无 filters | ✅ | ✅ | ❌ | ✅ 22条 | filters 非必需 |
| 乱码 x-s | ❌ | ✅ | ✅ | ❌ 300011 | x-s 格式校验 |
| 无 x-s/x-t | ❌ | ❌ | ✅ | ❌ 300011 | 签名必需 |

### 3.2 🔴 签名与请求体的绑定关系（XYS_ 格式）

> **注意**：以下测试基于旧 XYS_ 格式签名。XYW_ 格式的绑定行为不同，见 2.6 节。

| 测试 | 结果 | 结论 |
|------|------|------|
| 改关键词 + 原始签名 | ✅ 成功 | **XYS_ x-s 不与 body 绑定** |
| 改 page + 原始签名 | ✅ 成功 | **XYS_ x-s 不与 page 绑定** |
| 用当前时间戳替换 x-t | ✅ 成功 | **x-t 不严格校验时效** |
| 翻页（page 2,3） | ✅ 成功 | 签名可跨页复用 |
| 5个关键词用同一签名 | ✅ 全部成功 | 签名可跨关键词复用 |
| 用 search 签名调 feed | ❌ code=-1 | **x-s 与 API 路径绑定** |

**XYW_ 格式的绑定差异**（通过 payload 分析确认）：
- XYW_ 签名 **绑定请求体**：`_webmsxyw(path, body)` 中不同 body 产生完全不同的 payload
- XYW_ 签名 **绑定 API 路径**：不同路径产生不同 payload（与 XYS_ 一致）
- XYW_ 签名 **非幂等**：相同输入每次调用结果不同（前192字节确定，后16字节随机）

### 3.3 核心结论

```
请求头检测（新增发现）:
  ✅ x-b3-traceid: 随机 hex（8字节），每请求生成
  ✅ x-rap-param: 抓包值可静态复用（不与请求体绑定）
  ✅ x-xray-traceid: 随机 hex（16字节），每请求生成
  ❌ 缺少这些头 → cookie 被服务端标记（code=-100）

API 版本检测（新增发现）:
  ✅ 搜索必须用 so.xiaohongshu.com/api/sns/web/v2/search/notes
  ❌ 使用 edith.xiaohongshu.com/api/sns/web/v1/search/notes → cookie 被标记
  ✅ body 必须包含 message_id: "sending" 字段

x-s 和 x-s-common (XYS_ 格式):
  ✅ 与 API 路径绑定（search 的签名不能用于 feed）
  ❌ 不与请求体绑定（改关键词/页码不影响）
  ❌ 不严格校验时效（x-t 可用当前时间）
  ✅ v2 API 路径的签名可跨关键词/页码批量复用

300011 错误码:
  实际含义: "签名校验失败"（缺少 x-s-common 或 x-s 无效）

Cookie 标记行为:
  根因: 使用错误 API 版本 + 缺少关键请求头
  正确方案: v2 API + 全部浏览器请求头 → 60次请求不标记
```

### 3.4 批量采集实测结果（✅ 2026-07-10 验证）

**方式一：XYS_ 静态签名复用 + v2 API（✅ 已验证，推荐）**
- API 端点：`so.xiaohongshu.com/api/sns/web/v2/search/notes`
- 请求头：需包含全部浏览器请求头（x-b3-traceid, x-rap-param, x-xray-traceid, priority, sec-fetch-*）
- 请求体：需包含 `message_id: "sending"` 字段
- Cookie：需包含 `unread` cookie
- 签名复用：同一 API 路径的 x-s + x-s-common 可跨关键词/页码批量复用
- **实测结果**：20关键词 × 3页 = 60次请求，全部成功，0错误，1332条笔记，cookie 全程未标记

| 指标 | 结果 |
|------|------|
| 总请求数 | 60 |
| 成功请求数 | 60 |
| 失败请求数 | 0 |
| 总笔记数 | 1332 |
| 采集耗时 | ~2分钟 |
| cookie 标记 | ❌ 未标记 |
| 每页笔记数 | 22-25条 |

**方式二：XYW_ 动态签名生成（⚠️ 不可用）**
- 使用 `webmsxyw-node.js` 为每个请求动态生成 XYW_ 格式 X-s
- 每次签名 ~70ms，支持任意 API 路径
- **实测结果**：XYW_ 格式仍需要 x-s-common（返回 300011），XYW_+XYS_ 混合格式被拒绝（HTTP 461）
- 结论：XYW_ 无法替代 XYS_ 使用，需先破解 x-s-common 生成算法

**生成算法**：

```javascript
// 1. 构建 payload 对象
const en = {
  s0: 5,              // 固定值
  s1: '',             // 空字符串
  x0: '4.3.7',        // 指纹版本
  x1: 'web',          // 平台标识
  x2: 'PC',           // 设备类型
  x3: 'xhs-pc-web',   // 应用 ID
  x4: '6.31.2',       // webBuild 版本
  x5: a1_cookie,      // a1 cookie 值
  x6: mnsToken,       // X-Mns 相关（可空）
  x7: xsSign,         // X-Sign 头（可空）
  x8: fingerprint,    // localStorage 指纹（可空）
  x9: tb(mnsToken + xsSign + fingerprint),  // CRC32 变体哈希
  x10: sigCount,      // 签名计数
  x11: 'normal',      // 固定值
  x12: dsl,           // window._dsl（可空）
};

// 2. JSON 序列化 → UTF8 编码 → 自定义 Base64
const x_s_common = b64Encode(encodeUtf8(JSON.stringify(en)));
```

**关键函数**（均从 webpack 模块 41439 提取）：

| 函数 | 模块导出名 | 作用 |
|------|-----------|------|
| `encodeUtf8(str)` | `lz` | `encodeURIComponent` → 字节数组 |
| `b64Encode(bytes)` | `xE` | 自定义字母表 Base64 编码 |
| `tb(data)` / `w` | `tb` | CRC32 变体（多项式 0xedb88320，返回 `(-1 ^ crc) ^ polynomial`） |
| `crc32(str)` | `kn` | 标准 CRC32 |

**局限**：签名会过期（约数小时~数天），需定期重新抓取。

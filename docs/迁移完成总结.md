# Chrome Extension → Electron 迁移总结

## 迁移概述

将 chrome-extension 的全部核心功能（AI自主决策流程、上下文窗口管理、工作记忆等）迁移到 Electron 桌面应用。

**迁移时间**: 2026-07-07
**迁移文件数**: 22 个服务文件 + 1 个服务管理器 + main.js/preload.js 集成
**总代码量**: 约 12,000+ 行

## 核心技术适配对照表

| Chrome Extension | Electron | 说明 |
|------------------|----------|------|
| `chrome.storage.local` | JSON 文件（StorageService） | userData 目录下 storage.json |
| `chrome.storage.session` | JSON 文件（StorageService） | 同上，session 级数据 |
| `IndexedDB` | JSON 文件（DBService） | 每个 store 独立文件 |
| `chrome.scripting.executeScript` | `webContents.executeJavaScript` | 直接执行 JS 代码字符串 |
| `chrome.tabs.sendMessage` | `webContents.executeJavaScript` | 直接执行页面内容提取 |
| `chrome.tabs.captureVisibleTab` | `webContents.capturePage()` | 返回 dataURL |
| `chrome.tabs.get(tabId)` | `tabManager.tabs.get(tabId)` | 获取 BrowserView |
| `chrome.runtime.Port` | IPC 事件（sendEvent 回调） | `event.sender.send()` |
| `chrome.alarms` | `setInterval` | Electron 主进程持久运行 |
| ES modules (import/export) | CommonJS (require/module.exports) | 全部转换 |
| Service Worker 心跳 | 不需要 | Electron 主进程不会被随机终止 |
| `crypto.subtle` HMAC | `crypto.createHmac` | Node.js 原生 crypto 模块 |
| Shadow DOM 穿透 | `executeJavaScript` 注入 `_dqsa` | 深查询函数自包含 |

## 迁移的核心功能

### 1. AI自主决策流程（8阶段主循环）

1. **初始化与配置加载** - 域名策略、Agent配置、maxRounds计算、WorkingMemory/ContextCompressor/ScratchpadService 初始化
2. **历史清理与系统提示词构建** - 移除失败回复、清理孤儿消息、压缩历史
3. **自动页面感知与上下文注入** - 自动读取页面内容、搜索脚本、注入上下文
4. **简单请求快速路径** - 检测纯数据操作，跳过页面探索
5. **主循环** - 轮次检查、收敛提示、待办进度、WorkingMemory上下文、硬性规则检查
6. **LLM API调用** - 重试策略、降级处理、超时控制
7. **工具执行分发** - finish_task/search_tools/create_todo/DOM工具(16个)等
8. **循环结束处理** - 达到上限处理、保存最终摘要

### 2. 15层防死循环机制

1. 轮次硬上限（max 100）
2. 工具调用上限（max 300）
3. 全局超时（10分钟）
4. 连续无进展强制finish（5次）
5. 连续脚本失败强制finish（3次）
6. 收敛提示（70%/85%预算）
7. 中止信号检测
8. 工具幻觉拦截
9. 重复选择器提示
10. 中止信号传播
11. 纯文本工具调用检测
12. 预算预警导航
13. 域名安全策略
14. API降级重试
15. 上下文压缩

### 3. 上下文窗口管理

- **WorkingMemory**: 7类记忆（任务目标/当前页面/发现/决策/排除方案/数据引用/错误），FIFO淘汰 + 去重
- **ContextCompressor**: LLM驱动压缩 + 规则回退双轨设计
- **ScratchpadService**: 每轮覆盖式保存工作记忆状态，支持断点续传
- **PayloadStore**: 大数据存储+小摘要返回，8MB配额管理

### 4. 工作记忆

- 任务目标（截200字符）
- 当前页面（URL/标题/摘要截300字符）
- 关键发现（最多20条）
- 已做决策（最多15条）
- 排除方案（最多10条）
- 数据引用（最多15条）
- 错误记录（最多10条）
- 自动提取：从7类工具结果自动记录

## 新增 API 接口（preload.js）

渲染进程可通过 `window.api` 访问所有新功能：

| API 命名空间 | 功能 | 关键方法 |
|-------------|------|----------|
| `window.api.agent2` | Agent v2 自主决策 | start/abort/getStatus/onEvent/onDone |
| `window.api.config` | 配置管理 | getAI/saveAI/getSync/getAgent/getAppSettings |
| `window.api.todo` | 待办调度 | getTemplate/submit/getProgress/getContext |
| `window.api.scheduledTask` | 定时任务 | list/create/update/delete/enable/disable |
| `window.api.taskTemplate` | 任务模板 | list/create/instantiate/export/import |
| `window.api.toolRecording` | 工具录制 | list/get/delete/export/import |
| `window.api.scratchpad` | 中间推理 | list/load/delete/export |
| `window.api.humanIntervention` | 人工介入 | getPending/respond/cancel |
| `window.api.output` | 结果输出 | list/get/delete/export |
| `window.api.taskArchive` | 任务归档 | list/get/search/findSimilar |

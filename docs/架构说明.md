# AI Browser 插件架构说明

## 一、架构概览

AI Browser 采用**三层记忆 + 三阶段串行执行**架构，实现 AI 自主决策的浏览器自动化任务执行。

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           AgentService（入口）                                │
│  文件: background/services/agent-service.js                                  │
│  职责: Port 绑定、任务启动、状态管理、sessionId 分配、数据继承                │
└──────────────────────────────────┬──────────────────────────────────────────┘
                                   │
                                   ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                          AgentRunner（主循环）                                │
│  文件: background/services/agent-runner.js                                   │
│  职责: LLM API 调用、工具执行、上下文注入、阶段切换、消息压缩                  │
└──────────────────────────────────┬──────────────────────────────────────────┘
                                   │
                    ┌──────────────┼──────────────┐
                    ▼              ▼              ▼
┌───────────────────┐  ┌───────────────────┐  ┌───────────────────┐
│  WorkingMemory    │  │ TodoScheduler     │  │ PayloadStore      │
│  工作记忆层       │  │ 待办调度引擎      │  │ 数据暂存层        │
│  记录发现/决策/排除│  │ 三阶段切换规则    │  │ 大结果存储        │
│  阶段切换完整继承  │  │ 工具白名单控制    │  │ sessionId 隔离    │
└───────────────────┘  └───────────────────┘  └───────────────────┘
         │                      │                      │
         └──────────────────────┼──────────────────────┘
                                ▼
                    ┌───────────────────┐
                    │ GlobalDataStore   │
                    │ 跨阶段数据传递     │
                    │ dataOutputKey索引 │
                    └───────────────────┘
```

## 二、三层记忆架构

### Layer 1: Conversation（短期记忆）

| 属性 | 说明 |
|---|---|
| **存储位置** | `agent-runner.js → messages[]` |
| **生命周期** | 单任务运行期（阶段切换时部分清空） |
| **上限** | 40 条消息 / 8000 字符预算 |
| **压缩机制** | LLM 语义压缩（替代规则截断） |
| **阶段切换** | `messages.length = 0`，注入交接摘要 |

每轮 API 请求的结构：
```
[
  { role: "system", content: 系统提示词（阶段定义 + 工具列表） },
  { role: "system", content: 交接摘要 / WorkingMemory.toContext() },
  { role: "system", content: TodoScheduler.getStageContext() },
  { role: "system", content: 收敛提示（70%/85% 预算） },
  ... 历史对话（压缩后最多40条）,
  { role: "user", content: 用户消息 + 页面内容 + 数据摘要 },
]
```

### Layer 2: WorkingMemory（工作记忆）

| 属性 | 说明 |
|---|---|
| **存储位置** | `working-memory.js → state` |
| **生命周期** | 单任务运行期，**阶段切换完整继承** |
| **字段** | taskGoal / currentPage / discoveries / decisions / excluded / dataRefs / errors / stageHistory |
| **自动提取** | 工具结果自动提取关键信息 |
| **注入方式** | 每轮作为 system 消息注入 |

核心字段说明：
```javascript
{
  taskGoal: '采集当前新闻页面，内页也要采集',
  currentPage: { url: 'https://news.66wz.com', title: '温州新闻网', summary: '...' },
  discoveries: ['页面有3个tab，数据在tab2', '列表共15条新闻'],
  decisions: ['排除分页采集，只取首页', '选择器dl a匹配成功'],
  excluded: ['选择器.news-item无结果', '页面无分页按钮'],
  dataRefs: [
    { key: 'links', storeId: 'p3', count: 15, summary: '15条新闻URL' },
    { key: 'details', storeId: 'p8', count: 15, summary: '15篇正文' }
  ],
  errors: [{ tool: 'inject_script_10', error: '脚本不存在', round: 12 }],
  stageHistory: [{ from: 1, to: 2, reason: '4轮无进展', round: 8 }]
}
```

### Layer 3: Data Store（数据层）

**PayloadStore**：
| 属性 | 说明 |
|---|---|
| **存储位置** | `payload-store.js → entries[]` |
| **生命周期** | 内存，支持跨 session 继承 |
| **上限** | 30 条，FIFO 淘汰 |
| **触发条件** | 工具结果 > 1500 字符 |
| **索引** | sessionId + entry_id（如 `p3`） |
| **继承机制** | 新任务自动继承 5 分钟内的数据 |

**GlobalDataStore**：
| 属性 | 说明 |
|---|---|
| **存储位置** | `global-data-store.js → entries(Map)` |
| **生命周期** | 单任务运行期，阶段切换不清空 |
| **索引** | dataOutputKey（如 `links`、`details`） |
| **用途** | 待办数据依赖校验 + 跨阶段数据传递 |

## 三、三阶段串行执行架构

### 阶段定义

```
Stage 1（DOM 工具）        Stage 2（脚本处理）        Stage 3（结果汇总）
┌─────────────────────┐   ┌─────────────────────┐   ┌─────────────────────┐
│ read_page_content   │   │ search_tools        │   │ recall_data         │
│ extract_content     │   │ inject_script_N     │   │ finish_task         │
│ click_element       │   │ recall_data         │   │                     │
│ navigate_to         │   │ read_page_content   │   │                     │
│ ...                 │   │ finish_task         │   │                     │
│                     │   │                     │   │                     │
│ ❌ inject_script_*  │   │ ❌ click/fill/...   │   │ ❌ 所有执行工具      │
└─────────────────────┘   └─────────────────────┘   └─────────────────────┘
        │                        │                        │
        │   4轮无进展             │    3次脚本失败          │
        └────────────────────────┴────────────────────────┘
```

### 阶段切换规则

| 触发条件 | 动作 |
|---|---|
| Stage 1 连续 4 次无进展 | 系统强制切换到 Stage 2 |
| Stage 2 连续 3 次脚本失败 | 系统强制切换到 Stage 3 |
| AI 请求超 MAX_AI_REQUESTS(15) | 强制终止 |
| 工具调用超 MAX_TOOL_CALLS(30) | 强制终止 |
| Port 断开 | 立即终止 |

### 阶段切换时的上下文处理

```
1. WorkingMemory.addStageSwitch(from, to, reason, round)  ← 记录切换事件
2. ContextCompressor.generateHandoff()                    ← LLM 生成交接摘要
   ├─ 成功：{ role: "system", content: "## 前序阶段成果..." }
   └ 失败：回退到 WorkingMemory.toHandoffContext()
3. messages.length = 0                                    ← 清空对话历史
4. 注入：[新systemPrompt, 交接摘要system, userMsg+dataSummary]
5. WorkingMemory 完整继承                                 ← discoveries/decisions/excluded 不清空
```

## 四、数据流转全景

```
用户消息 → AgentService.startAgent()
  │
  ├─ 1. 设置 sessionId = `s_${tabId}_${timestamp}`
  ├─ 2. PayloadStore.inheritFromLastSession(newSessionId, 300000)  ← 继承5分钟内数据
  ├─ 3. TodoScheduler.clear()
  ├─ 4. WorkingMemory.init(sessionId, userMessage, pageContent)
  │
  └─ 进入 AgentRunner 主循环（每轮）
       │
       ├─ checkPortConnected() → 断开则终止
       │
       ├─ 注入上下文：
       │    ├─ WorkingMemory.toContext()（阶段切换后前2轮只注入errors/excluded）
       │    ├─ TodoScheduler.getStageContext()（Stage3才注入全局存储）
       │    └─ 收敛提示（70%/85% 预算）
       │
       ├─ LLM API 调用 → 返回 tool_calls 或 finish_task
       │
       ├─ 工具执行循环：
       │    ├─ 工具白名单校验（幻觉拦截）
       │    ├─ 执行工具 → toolResult
       │    ├─ WorkingMemory.autoExtractFromToolResult()  ← 自动提取
       │    ├─ 结果处理：
       │    │    ├─ >1500字符 → PayloadStore.add() + WorkingMemory.addDataRef()
       │    │    ├─ dataOutputKey → GlobalDataStore.set()
       │    │    └─ 否则 → smartTruncateResult()
       │    ├─ TodoScheduler.matchToolCall() → 进度追踪
       │    └─ toolResult 推入 messages[]
       │
       ├─ messages.length > 40？
       │    ├─ ContextCompressor.compress()  ← LLM 压缩前60%消息
       │    └ 失败：回退到规则驱动压缩
       │
       ├─ shouldSwitchStage()？
       │    ├─ WorkingMemory.addStageSwitch()
       │    ├─ ContextCompressor.generateHandoff()
       │    ├─ messages.length = 0
       │    └─ 注入交接摘要 + WorkingMemory 继承
       │
       └─ finish_task → chatHistory持久化 → PayloadStore清理旧数据
```

## 五、上下文压缩机制

### LLM 驱动压缩

**触发条件**：messages 超过 40 条

**压缩流程**：
```
1. 取前 60% 消息（跳过 system prompt）
2. 构建 LLM 输入：消息流文本 + WorkingMemory.toContext()
3. 调用 LLM API（temperature=0.1, max_tokens=1024）
4. 输出结构化摘要：
   ## 关键发现
   ## 已做决策
   ## 排除方案
   ## 数据引用
   ## 当前状态
5. 摘要替代 cutOff 之前的所有消息（一条 system 消息）
```

**成本**：每次压缩消耗 1 次 LLM 调用

**回退机制**：LLM 压缩失败时，使用规则驱动压缩（保留关键工具结果的摘要）

### 阶段交接摘要

**触发时机**：阶段切换前

**生成流程**：
```
1. 收集前序阶段的关键信息
2. 调用 LLM API（temperature=0.1, max_tokens=800）
3. 输出结构化交接：
   ## 任务目标
   ## 前序阶段成果
   ## 关键发现
   ## 数据清单（含精确 recall_data ID）
   ## 需要注意的问题
```

**注入规则**：
- 阶段切换后前 2 轮：交接摘要已包含发现/决策，WorkingMemory 只补充 errors/excluded
- 超过 2 轮后：注入完整 WorkingMemory（跳过 currentPage 避免与 userMessage 重复）

## 六、工具执行与结果处理

### 工具白名单校验

```javascript
// Stage 1: inject_script_* 全部拒绝
if (toolName.startsWith('inject_script_') && currentStage === 1) {
  return { error: 'Stage 1 不允许调用远程脚本工具，请使用 DOM 工具探索页面' }
}

// Stage 2: DOM 工具部分拒绝
if (!STAGE2_TOOLS.has(toolName) && !toolName.startsWith('inject_script_')) {
  return { error: `Stage 2 优先使用 inject_script_* 工具，${toolName} 不在白名单` }
}

// Stage 3: 所有执行工具拒绝
if (toolName !== 'recall_data' && toolName !== 'finish_task') {
  return { error: 'Stage 3 只能调用 recall_data 和 finish_task' }
}
```

### 结果截断与存储

```javascript
// 判断是否需要存储到 PayloadStore
if (toolResultStr.length > 1500 || hasDataOutputKey) {
  const summary = generateSummary(toolResult, 150)
  const storeId = payloadStore.add(toolName, toolResult, summary)
  workingMemory.addDataRef(key, storeId, count, summary)
  
  if (hasDataOutputKey) {
    globalDataStore.set(dataOutputKey, toolResult)
  }
  
  return { summary: `${summary} [完整数据存于 ${storeId}]` }
}

// 否则智能截断
return smartTruncateResult(toolResult, 300)
```

### recall_data 工具

**查询 PayloadStore**：
```javascript
// 默认只查当前 session
payloadStore.query({ entry_id: 'p3' })  // → 返回完整数据（限制5000字符）

// 跨 session 查询（显式指定）
payloadStore.query({ entry_id: 'p3', crossSession: true })
```

**查询 GlobalDataStore**：
```javascript
// 返回摘要（>1500字符自动存入 PayloadStore）
const data = globalDataStore.get('links')
if (data.length > 1500) {
  const storeId = payloadStore.add('recall_data', data, summary)
  return { summary: `${summary} [完整数据存于 ${storeId}]` }
}
return data
```

## 七、端口连接状态监控

### 检测机制

```javascript
// 每轮循环开始时发送心跳
checkPortConnected(tabId) {
  try {
    port.postMessage({ type: 'heartbeat' })
    return true
  } catch {
    port = null
    console.log('[AgentService] Port 已断开，任务将被终止')
    return false
  }
}
```

### 断开处理

```
用户关闭 SidePanel → Port 断开
  │
  ├─ 下轮循环开始：checkPortConnected() 返回 false
  ├─ 立即终止任务（不再继续 API 请求）
  ├─ 清理资源（不持久化中间数据）
  └─ 日志记录：[Agent] Port 断开终止
```

## 八、任务终止与错误恢复

### 正常终止流程

```
finish_task
  │
  ├─ WorkingMemory 总结发现/决策/数据引用
  ├─ 输出最终 summary
  ├─ chatHistory 持久化（保留最近50条，截断至8000字符）
  ├─ PayloadStore 清理超过 5 分钟的旧数据
  └─ Port 解绑
```

### 异常终止处理

| 异常类型 | 处理方式 |
|---|---|
| Port 断开 | 立即终止，不持久化 |
| API 超时 | 等待 TIMEOUT_MS(10分钟)，超时则终止 |
| 工具执行超时 | 等待 ACTION_TIMEOUT_MS(1分钟)，超时则返回错误 |
| LLM 压缩失败 | 回退到规则驱动压缩 |
| 阶段交接失败 | 回退到 WorkingMemory.toHandoffContext() |

### 断点续传（AgentResumeService）

**快照内容**：
- userMessage（原始用户请求）
- todoState（待办状态）
- currentStage（当前阶段）
- workingMemory.state（工作记忆）
- payloadStore sessionId

**限制**：不保存 messages[]，续传时从零重建对话

## 九、与现代 AI 编程工具的对比

| 维度 | AI Browser | Cursor/Claude Code | Devin |
|---|---|---|---|
| **数据持久化** | 内存（PayloadStore），支持跨 session 继承 | 文件系统 | Workspace 文件 |
| **上下文加载** | 全量对话历史 + LLM 压缩 | 用户显式引用 @file | 按相关性加载 |
| **语义检索** | ID 寻址（recall_data entry_id） | 向量索引 + AST 索引 | 文件搜索 |
| **工作记忆** | WorkingMemory（结构化） | 隐式（diff hunks） | 显式 scratchpad |
| **长期记忆** | project_memory.md（不自动注入） | .cursorrules（自动注入） | Playbook |
| **错误恢复** | 快照（元数据 only） | git rollback | 环境快照 |
| **阶段控制** | 系统驱动（AI 无切换权） | 用户驱动 | Devin 控制 |
